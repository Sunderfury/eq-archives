斜め走り

CTRLを押しながら旋回しようとすると、方向を変えずに左右に移動する事ができます。
Auto Run状態(DefaultではNum Lockです)でこの操作を行うと、走りながら斜めに滑るような移動になるため、通常のRunよりも滑った分だけ移動距離が伸び、より速く走っている事になります。
長距離を移動する場合や敵から逃げる場合に覚えておくと便利です。
また左右の水平移動は、戦闘中に細かい立ち位置を調整する場合にも有効です。
キーをカスタマイズした場合は、この限りではありませんので悪しからず(泣)


弓、投擲武器の使い方

弓はAuto Attack状態では使えません。
弓をRange Slot、矢をAmmo Slotに入れて、Range AttackをHotKeyに登録すれば準備完了です。
弓と矢には種類に応じてRangeが設定されていて、弓と矢のRangeを合計した値が矢の最大到達距離になります。
自分と目標の距離が矢の最大到達距離を越えていると、攻撃することができません。
また距離が近すぎても攻撃することができません。
Range Attackボタンを押す度にTargetに矢が放たれ、Ammo Slotの矢を消費していきます。
Ammo Slotの矢が残り1本になると、荷物の中に同じ種類の矢があれば、それを消費します。
同じ種類の矢が荷物から無くなると、Ammo Slotの最後の1本を放ちます。
矢は自分とTargetの間に物理的な障害がある場合(視線が通らない場合、高低差等)には命中しません。
また、矢を放つ時にTargetとの間に別のNPC等が割り込んだ場合、そのNPCに命中します。
投擲武器の場合は、Range Slotに入れます。
使い方は弓とほぼ同様ですが、投擲の場合はRange Slotの武器自体を消費します。
投擲された武器は命中の成否に関わらず失われます。
投擲可能な武器を投擲目的以外で使う場合(Resist上昇効果を期待して装備する等)は、誤って投げてしまわないように注意してください。


Bash

BashはKickやSlamと併用できないSkillで、盾を持っている場合に盾で殴りつけて攻撃するSkillです。
Bashが命中すると、高確率で相手をStunさせて一定時間行動不能にさせると共に、相手がSpellをCastしていた場合は中断させる事ができます。
特に相手がHealer Typeの場合は、Healされると戦闘が長引いて危険なので、相手がSpellのCastを始めたらBashを狙ってみると良いでしょう。
ただし、Lady Vox等の一部のMobやLV55を超えるMobはStunによるSpellの中断は起こりません。
LV13でDual Wieldを使えるようになると、常時盾を持つ事はなくなります。
Slamを持たない中・小型種族の場合は、自然とKickを多用するようになるでしょう。
しかし、BashはKickよりもTaunt効果が強く、上記の効果と併せてSlamを持たない種族の場合はLV55になるまで有用なSkillとなります。
戦闘中の操作が煩雑になり、かつ盾を置くために貴重なGeneral Slotを消費しなくてはなりませんが、マメに使うとStun効果で大きく被Damageを押さえる事もできます。
Slamを使えない中小型種族でSoloを強いられた場合、この方法を試してみるとかなり楽になるはずです。
だからと言って必須なわけでも無いので、こういった作業を苦にしない方、あるいはBash Skillを育てたいSkill Collectorな方はぜひ。
どうしてもGeneral Slotを使いたくない場合、盾の入っているBagをHot Keyに置いて代用することもできます。
この場合はBagを開いておかないと緊急時にカーソルに武器や盾が貼り付いたまま対処しなくてはならず、不便度は当社比2倍です(泣)

①左手のMelee Slotと、InventoryのGeneral Slot 8個のいずれか1つをHot Keyに登録。
　登録したGeneral Slotに盾を置きます。


②戦闘中は盾をHot Keyから拾い上げてHoldしておき、Bash可能(ボタンが戻る)になったら素早く左手のMelee Slotの武器と盾を入れ替えてBash、Bash後再び左手のMelee Slotの盾と武器を入れ替えます。
　Add等で急なTargetの必要がある場合は、Holdしている武器か盾を素早くHot KeyのGeneral Slotに押し込みましょう。
　慣れないうちは武器や盾を落とす事も有るので、Item DropのオプションでDrop不可にしておくと良いでしょう。



AC is all

ACはArmor Classの略で、物理攻撃を受けた場合に働きます。
ACには「回避に使われるAC(SkillやAGI等に依存)」と「Damage軽減に使われるAC(鎧のAC等に依存)」が存在し、画面上にはこの2つのAC値に依存した値が表示されます。
ACが上昇すると、攻撃を回避しやすくなったり、受けるDamageを軽減しやすくなったりします。
余談ですが、WarriorのACは一般に鎧に依存する度合いが大きい為にDamage軽減へ寄与し、MonkのACは回避Skillに依存する度合いが大きい為にDamageそのものを受けにくい傾向があります。
よって、通常はWarriorが最も重視すべきなのはACです。
ではACをトコトン上昇させると凄いTankになるのかと言えば、実はそうでも無かったりします。
ACには俗にCapと呼ばれる限界値が存在し、その値に近づくと上昇したAC分に対する効果が薄れてきます。
色々な実験結果が出ていますが、SoLでの拡張時点の通説では実用的なACは1250～1350で、Arch Type AbilityのCA(Combat Agility)とCS(Combat Stability)に依存するようです。
装備とSkillを整えてこの値に達した場合は、HPを増やす方向になっていきます。
AGIが75を下回るとACが極端に下がり始めるので、ある程度は補強しておくと良いでしょう。
余裕が無いと、Berserk～瀕死の場合はAGIが低下する為に、75を維持し難くなります。


HPのあれこれ

HPが2BBL(40%)を下回ると、多くの場合、敵から集中的に狙われます。
身内では死に掛けTaunt等と呼んでいますが、この状態で敵から攻撃を受けている場合、Tauntが通用しません。
剥がす為にはDivine Aura等の敵の攻撃を無効化する魔法、あるいはHealingでHPを2BBL以上に回復させなくてはなりません。
Low HPでDA状態になってしまった場合等は、思い切ってHP Buffを消して見かけのHPを2BBL以上にするのも一つの手です。


SoloでCasterを相手にする場合

Caster Typeを相手にSoloを強いられた場合、Mobが詠唱を開始したら素早く背後へ回り込んでBashします。
Cast中の背後からのBashは高確率で詠唱中断を誘える上に、Cast Timeが経過するまで、彼らは行動する事ができません。
この間にわずかながら、彼の無防備な背中に攻撃を加える事が出来ます。
機会は何度も訪れるので、積み重ねると目に見えて戦闘が楽になるでしょう。
Slamを持たないRaceでBashを使いこなすにはこちらを参考に。


Pull

EQに存在するMobは大きく分けると固定、巡回、ウロウロ君(泣)の3Typeに分かれます。
固定はPopした場所から基本的に動かないMobです。
巡回は規定の場所にPopした後、決まったルートを移動し続けます。
Dungeonに居るMobはほぼ上記の2Typeです。
屋外ではPopした後適当にZone内を歩き回るウロウロ君が大半を占めます。

固定型が数体まとめて配置されている場合、勢いでPullすると酷い目に遭う事も多いので、以下の点に気をつけて挑戦してみてください。

・巡回するMobが周辺に居ないか
　Linkする可能性のある位置に巡回が来るかどうかを事前に観察します。
　巡回が居る場合は、固定型とLinkしない位置で先にPull、あるいはLinkしない位置に移動した隙に固定をPullします。
　固定型だと思われる集団に実は数体の巡回が混ざって居た、などという事もあるので、余裕がある場合はジックリと観察してみてください。

・LevelとTypeをCheck
　Pullする時は、基本的に集団の中で最も高いLevelのMobを狙います。
　Pullする人と相対的にLevelの低いMobは、Levelの高いMobよりもLink範囲が狭いので、Linkしない可能性もあるからです。
　Pullした人のLevelにも依存するので、場合によってはParty内の最もLevelの高い人(Casterでも)にPullしてもらうのも一つの手です。
　MobにCasterが含まれる場合、Pull後にMobの視線が通らない場所へ隠れる事をお勧めします。
　NPCのCastは視線が通らなくても効果を発揮しますが、視線が通らない場所ではCastを開始しません。
　その為、PullerがPartyの近くの視線が通らない場所に駆け込んでしまうと、すんなりPartyの場所まで走ってきます。
　隠れる場所が無い場合は仕方ありませんが、ボーっとしているとDispellされたり、Castしていて動かない間に巡回がAddしたりする事も良くあります。
　Warrior TypeとCaster Typeが混在している場合は、基本的にWarrior Typeを狙います。
　Pull中のSpell Castによる様々な弊害を避けるためですが、後述するLullを使う場合は特にCasterを後回しにしてください。
　延々とSpell Castされると、その間にLullの効果が切れてしまい、結局Linkされてしまいます。

・HarmonyとLull
　Harmonyは屋外(変更後のCazicとPoP Zonesを除く)限定でRangerとDruidが使用可能な魔法で、Pullの最強の補助魔法です。
　このSpellは、効果を受けたMobのLink範囲を極めて小さくします。
　LinkさせたくないMobにこの魔法を掛ける事で、ほぼ100%、Linkを切る事ができます。
　Resistされないのも大きな特徴です。
　似た効果を持つSpellにLull系があります。
　このSpellも効果を受けたMobのLink範囲を狭めますが、比較的短時間で効果が切れてしまい、Resistされる事もあるので使い難い印象があります。
　実際に使用する場合は、LinkさせたくないMobにLullを掛け、素早くPullしたいMobをPullする事になります。
　LullがResistされた場合、あるいはPullが遅れた場合は、多分酷い目に遭うと思われます(泣)
　※PoP Zonesでは追加されたHarmony of Natureが使えますが、こちらはAoEでは無くSingle Targetで、かつResistされることがあります。

高低差や障害物等で飛び道具が使えない場合にBashを使ってPullすると、Stunに成功すれば走って距離を稼ぐ事が出来ます。
Pull中のMobからのMelee Damageを押さえる点でも盾を持ってのPullは有効です。
Chardok等のPullが長くなりがちなZoneでは、Mobに追いつかれるとPartyの元に辿り着くまでにかなりのDamageを受けます。
追いつかれたら落ち着いてBashして、Stunさせてから再び走りましょう。
Mobの追跡経路は適時再計算される為に、場合によっては思いもよらない方向へ突然回り込んでADDを招く事もあるので注意します。
特に建物などの障害物が多いZoneはこの傾向があります。
長射程の弓などで遠くからPullして引き離した場合、MobがHateを持ったまま追跡を断念する事があります。
Zoneしない限りはHateを持ち続けるので、運が悪いと忘れた頃にTrainになって襲って来ます。
極端に距離が離れたり、Stack等で強制的にMobの位置が戻ると、Hateがクリアされる場合もあります。


正しい殴り方、殴られ方

一般に敵の背後からの攻撃は、正面からの攻撃より有効です。
Mobに殴られる場合は背中を見せないように、Mobを上手く誘導します。
背後のCasterへ張り付いたMobを振り返って剥がした後、再び前を向いて元の敵を殴ると、剥がしたMobに自分の背中をさらしてしまいます、注意しましょう。
剥がしたMobとCasterと攻撃対象が常に視野に入るのが理想です。
剥がしよりもMeleeを主体とするTank陣は、Mobの背後から攻撃します。
背後からの攻撃はReposteされない為に、飛躍的にHit率が向上します。
無闇に沢山のTankが背後へ回るとMobを押してしまうので、Mobの正面に沢山のTankが居る場合に、バランスを考えて背後へ回ります。
Mobの周囲にTankがバランス良く散開すると、Mobを均等に押すため位置が変わり難く、位置を変えたい(押したい)場合にも素早く対応出来ます。
MobにDSが付いている場合は、無闇に殴り続けるとこちらも疲弊します。
Wizard系のDSは恐れる事も無いですが、Druid、Magician系のMobのDSは無視できないので注意が必要です。
DSに気づいたら、無闇に攻撃せずにDispell等を待つか、両手持ちの攻撃回数が少ない武器に変更しましょう。


寝かしと剥がし

まず、事前に寝るMobと寝ないMobの確認をしておきます。
寝ないMobはTankが捕まえて押さえる事になります。
ENCさんやBRDさん、NECさん等のMesmerize系によるTauntのサポートは、Knight Class等の方がWarriorよりも適任です。
下手に助けようとして殴ると、大抵の場合、より悪くなります(泣)
Mezが効きにくいMobをWarriorのサポートで寝かす場合は、まずはWarriorが殴ってHateを稼ぎ、その後Attack OffしてMez Workに入ります。
DoT効果のある武器を使っていると、Attack OffでもReposteしてProcする場合があるので注意してください。


Tauntのあれこれ

TauntはAll or NothingのSkillで、成功するとMobのHate ListのTopに自分を置く(=自分が殴られる)事ができます。
昔から「Tauntを連打するか否か」について議論されていますが、これもなかなか決着がつかないようです。
Tauntの成功率はかなり悪い(10回連続失敗もめずらしくはありません)ので、狙った時に上手にTauntできるわけでもありません。
※Patchでかなり改善されています
私の場合、Tauntを連打したくなるほど他のTankが殴られているケースでは、むしろSub Tank的に動くようにしています。
最近はKnight Classも丈夫になっているので、WarriorでなければMain Tankが務まらないケースは、さほど多くありません。
意外と2HanderでもTauntできたりするので、色々試してみてください。
なお、SitによるTauntはHate値に上乗せされるらしく、例えばMez状態のMobにTauntを試みて成功した場合でも、Hate値が同様に高いCasterがSitしていた場合はCasterが 殴られます。


Damage Bonus

Damage Bonusが得られるClassの場合は武器のStatusに明記されるようになりました。

右手に持った武器での攻撃に適用されるもので、左手の攻撃よりも大きなDamageを与える事ができます。
ちなみに、Damage Bonusがいくつなのかは、Critical Hit時に知る事ができます。
周囲のPCも見ることができるCritical Hit時に表示されるDamageは、Damage Bonusを含んでいません。
この表示の直後に実際に相手に与えたDamageが表示されますが、このDamageからCriticalのDamageを引いた値がDamage Bonusです。
計算式は

Damage Bonus = (LV - 25) / 3

で、端数切捨てです。
LV60で11PointのDamage Bonusになります。
両手持ちの武器の場合はまたちょっと違う式になります。

2H-FAST(DLY<=27):Damage Bonus = (LV - 25) / 3
2H-MID(28<=DLY<=39):Damage Bonus = (LV - 25) / 3 + (LV - 27) / 4 - 1
2H-SLOW(40<=DLY<=42):Damage Bonus = (LV - 25) / 3 + (LV - 27) / 4
2H-VERYSLOW(DLY>=43):Damage Bonus = (LV - 25) / 3 + (LV - 27) / 4 + (DLY - 34) / 3

端数切捨てです。
上記の通り、両手持ちの武器はそのDLY(Delay)によってDamage Bonusも変化し、振りの遅い武器ほど大きなDamage Bonusが得られるようになっています。
ではDLYが大きければ良い武器なのかと言うとそうでもなく、基本的に空振りが多く、Damageのばらつきも大きいEQでは、攻撃回数が多くなる武器(=DLYの小さな武器)の方が良い評価を得ます。
特に片手持ちの武器の場合は、Damageが大きな武器よりもDLYの小さな武器を右手に装備したほうが、Damage Bonus込みで攻撃力が増加します。
右手に持てば表示上よりも大きなDamageの武器になる、と考えてください。
蛇足ですが、1DLYは0.1秒になり、10DLYの武器は1秒に1回ずつ攻撃判定が発生します。

BreakとCamp

主にDungeonでPlayする場合、ある特定の場所に陣取り、そこからMobをPullするスタイルのCampが一般的に行われます。
Campする場所はItem狙いの場合はItemをDropするMobがPopする場所、EXP狙いの場合はPopや巡回が来ない安全な場所が好まれます。
安全な場所でCampする場合はあまり気にせずとも良いのですが、他のCamperとの兼ね合いでMobがPopする場所でCampする場合、あるいはPullしたい場所にMobが複数Popする場合は、その後のPullやCampを楽にする為にBreakとPop管理をします。
簡単な例を挙げると、A、B、Cの3体のMobが固定でPopする場所を攻める場合、まずAを倒します。
その後B、Cを倒すわけですが、Partyに余力がある場合、残ったBとCをなるべく「ゆっくり」倒します。
基本的に同じZoneのMobは同じRepop間隔を持つので、意図的に時間差を付けてMobを倒す事で、次回のRepopをコントロールする事ができます。
この例では、次回のPopではAがPopした後、BをKillするまでにずらした時間だけ遅れてBがPop、更に遅れてCがPopします。
こうするとAをKillした後に余裕を持ってB、Cを倒せるので、楽にCampを維持できるようになります。
これをBreak、と呼びます。
MobがPopする場所でCampする場合は、AをKillした後に再びAがPopするまでの時間を計っておきます。
B、CのPopを片付ければ、次のAがPopするまでの時間を他のMobのPullに使えることになりますし、万一事故が有って戦闘中にRepopが重なる場合でも、いつRepopするかを把握できていれば、逃げるかNukeで倒すか、などの判断がしやくなります。
安全な場所からのPullでも、これを気に掛けておけば、初回のPullでは沢山INCしてしまっても、2回目以降のPullが楽になります。
極めていくと効率重視になってしまうので、あまりキッチリしないほうが楽しく遊べるかも知れません。
頻繁に通うようになると、Pullの間隔やKillに掛かった時間などから勘で分かるようになって来ますが、初めて訪れるDungeonや戦力的に不足気味でCampしなければならない場合などは、気に掛けておくと良いでしょう。
特定の範囲のMobを忘れずにPullしておけば、Pullに掛かる時間の分だけRepopがずれていくので、あまり神経質にならなくても良かったりします(汗)


Defensive vs Evasive

以下はBoss級MobのTankingを行う場合、DefensiveとEvasiveのどちらがより有効かの指針です。
実際にはMobのDPSはいくつか、Slowが入るかどうか、MTはCAを取っているか、CCHの間隔を何秒にするか、できるか(CLRの数で限界がある)などの要素も絡んできます。
なおかつ運の要素も絡んできますので、Evasive使ってるのにFlurry4連打全部Hitされた、等の事態については知りません((((；-_-)

DB : Mobの持つDamage Bonus
DI : Mobの持つDamage幅の基礎値

DMG = DB + Random 20(1～20) * DI
最大DMG = DB + 20 * DI
最小DMG = DB + 1 * DI = DB + DI

DI = (最大DMG - 最小DMG) / 19
DB = 最小DMG - DI

DefensiveはDIを50% Cutする効果を持ちます。
Evasiveは回避率を増加させます(Dodge/Parry/Reposte=12%、敵のMiss=30%、Evasive時の敵のMiss=46%と仮定)
DIが大きなMobほどDefensiveの効果が大きく、DBが大きなMobほどEvasiveの効果が大きくなります。

例えばDtVの場合:
Derakor the Vindicator : DB=200 DI=25
Defensive時の最大DMG = 200 + 20 * (25 / 2) = 450
Defensive時の最小DMG = 200 + (25 / 2) = 213
Defensive時の平均DMG = (450 + 213) / 2 = 331
Evasive時の最大DMG = 200 + (20 * 25) = 700
Evasive時の最小DMG = 200 + 25 = 225
Evasive時の平均DMG = (700 + 225) / 2 = 462

Defensive時のEvasiveに対する平均DMGの比較 = 331 / 462 = 72 : 18% Cut
Evasive時のDefensiveに対する平均DMGの比較 = 42 / 58 = 78 : 22% Cut

vs DtVの場合はEvasiveが有効

例えばDoomshadeの場合:
Doomshade : DB=110 DI=15.5
Defensive時の最大DMG = 110 + 20 * (15.5 / 2) = 265
Defensive時の最小DMG = 110 + (15.5 / 2) = 118
Defensive時の平均DMG = (265 + 118) / 2 = 192
Evasive時の最大DMG = 110 + (20 * 15.5) = 420
Evasive時の最小DMG = 110 + 15.5 = 126
Evasive時の平均DMG = (420 + 126) / 2 = 273

Defensive時のEvasiveに対する平均DMGの比較 = 192 / 273 = 70 : 30% Cut
Evasive時のDefensiveに対する平均DMGの比較 = 42 / 58 = 78 : 22% Cut

vs Doomshadeの場合はDefensiveが有効
