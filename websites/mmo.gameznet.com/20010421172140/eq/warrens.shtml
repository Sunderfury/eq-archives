The Great Warrens Debate


So what is up with this new Warrens zone anyway? Ok, so it might sound like a bad Seinfeld opening line, but the question remains true. Verant has implemented this new looting code, which has become the nightmare of all high-level players since it was announced. Ok, so the code probably won't be put into any of the Everquest proper zones, nor any of the current expansion zones, however it begs the question as to whether the all-new-looting-code will be in the next Everquest expansion, if such a thing is ever released.

The code itself is quite simple: If nobody in the group which "gets the kill" gains experience from this kill, any rare loot that the monster has is thrown away. Furthermore, any NODROP, LORE or MAGIC items are thrown away. All good loot in Everquest has one of these tags on it, meaning that high levels can no longer get good items off monsters, which are deemed too weak for them. This seems like not a bad idea on the surface, but if you examine the implications of the code, you will see why I and so many other players have a problem with it.

First and foremost, it denies access to certain items for a large majority of the people in the game. For example, imagine if that rule was in effect in Qeynos Hills. This happens to be the zone where one of the oldest, still very useful items in the game drops. This item is the Glowing Black Stone, and it is dropped of a very rare level 16 necromancer named Pyzjin. What this means is that only characters from about levels 14 to 22 or so have any chance of being able to get this item. Sure you can argue that a low level group can kill it and a high level character can loot it, but all this does is make it so that high levels have to create characters, which cover multiple level ranges to camp items. It would not stop farming it would just make it a lot less convenient. On top of it all, the item in question is much too good for the level of the monster - perhaps this is the fault of Verant, but what has happened is that higher levels are always the ones, which kill the monster first.

A big problem that I have with the system is the "role playing" aspect of this code, or the logic of how it works if you will. For example if you had a nice, rare item on a level 20 monster, how can the monster not drop the sword it is hitting you with if you are above a certain level? It just simply doesn't make sense that the loot vanishes into thin air, just because you got no experience of for killing that said monster. In fact, if you are high enough level and kill the monster in one cast, how could the monster have any time to destroy the item before he/she/it passed on? The answer is that it doesn't make any sense. In a world that is supposed to be built around role-playing, a stunt like this seems to go against everything that it should. Everquest is the self-proclaimed founder of the MMORPG category of games, yet if this code becomes popular in the zones that Verant is creating, it takes the RP out of RPG.

Yet another problem with the said system is that it makes questing significantly more difficult. A large number of quest items are LORE, MAGIC or NODROP. The problem comes most seriously with the NODROP quest items. If this new code becomes commonplace, the whole quest system in the game will likely have to change. It will either be so that quest items will not be NODROP, MAGIC or LORE, in which case people will start selling quest rewards like mad, or they will have to start making exceptions to this rule. Furthermore, if they start making exceptions to the rules that are convenient for them, all of the reasons for having the system in the first place will start to fly out the window and not have the same validity they used to. It will start to look like Verant is just doing this, in a large part, to stop twinking - which many people (including myself) feel is a very valid way of playing.

This has been a short Editorial because there really isn't a whole lot to say about the issue at hand. Perhaps I just needed to express my opinions about the whole situation to the world, or perhaps I actually have some valid and reasonable points that the people over at Verant simply did not consider when making this new code. The point is that there are always two sides to every coin, and I wanted to make people aware of the side Verant is not showing everyone

--Adrian "Stranax/Thundak" Bloy
