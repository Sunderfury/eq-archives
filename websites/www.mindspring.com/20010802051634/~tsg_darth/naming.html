Naming Conventions - Last updated 7/3/01

   The Burnroom has implemented procedures in the past to ensure that the correct versions of all games are being burned, and tested. If this procedure is followed, it will help ensure that the correct version is being burned, whether for test or for submission. It will also help in accurate archiving of all versions. The naming convention has been revamped to help make the process as easy as possible. Your cooperation in adhering to this scheme would be appreciated.

All patches must be zipped. This will ensure the integrity of the patch.

Patch names must follow an 8+3 format, regardless of whether the patch is for a Win9x project or not. Any file may be transferred or manipulated on a DOS based machine. This is a limitation The Burnrrom is stuck with. If a patch not following an 8+3 format is handled by a DOS based machine, then truncation will occur, resulting in the undesired effect of renaming a file.

Imagine a file named 12345678.zip. Characters 1&2 will represent the Platform. Characters 3&4 will represent the Product Name. Character 5 will represent the Country. Characters 6 will represent a debug ro retail version and 7&8 will represent the version number. The version number is a decimal number starting at 01, and incremented by one each and every time a new version arrives, regardless of whether the versions are 5 minutes apart.

For instance, imagine the fifth version of WWF: Attitude, for North America, for the N64 arrives. The patch should be contained within a zip with the name, N4TIED05.zip. The naming convention has been modified to match the game title  as closely as possible.
