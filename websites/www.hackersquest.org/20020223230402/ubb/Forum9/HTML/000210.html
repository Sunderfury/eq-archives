I've posted the final version of AO Mission Display (v1.4) at my website... my 3 month subscription is almost up and I'm not renewing.

New in v1.4 ====
* Improved packet handling. There should now be much less chance of the last few missions being left out, or producing garbage.
* Added an 'Item Watch' function, to look for items containing specific text and play a sound or show a message when there is a match.
* Now properly identifies the find item for missions set to Left, Right, Right, Right, Middle, Left.

As this is the final version, I've also posted the full source code, to aid others in developing AO-related tools, or improve on AOMD. It is a MS VC++ project fully writen in C (not C++) using Win32API (not MFC).

Both can be downloaded from by website http://www.muzzleflash.da.ru

Rgds,
Muzzleflash.

quote:
Oh, and is anyone else havign problems with the watch feature?

Make sure you put each item on a separate line, not separated by commas... I probably should have made that more clear. Example:

battle suit
6k-x
omni-pol elite

Also if you are using the 'play sound' notification, it will not work with some soundcards... you'll need to switch it to the visual notification instead. It works fine with my SB Live, which has something like 1024 sound channels, but some cheapy soundcards only have a few sound channels, in which case AO probably has control of all of them, leaving none for AOMD to use.
