This is how to get the CVS version of ShowEQ working with decryption, assuming you have all correct library files installed. I may make a more comprehensive guide later that covers installing the libs, but I don't have time right now.

Note: if the wget command does not work for any reason, you can always open the files in your web browser of choice and save them. Also, you can get the patch from SourceForge at http://sourceforge.net/tracker/index.php?func=detail&aid=493581&group_id=10131&atid=310131 (for now).

cd /some/seq/directory
export CVSROOT=:pserver:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq
cvs login
(hit enter when it asks for a password)
If you have never checked out SEQ from CVS:
cvs -z3 co linux
If you have previously checked out SEQ from CVS:
cvs update linux
cvs logout
cd linux
wget http://zeer1.tripod.com/libEQ.a
mv libEQ.a /usr/lib/libEQ.a
wget http://zeer1.tripod.com/showeq.diff
patch -p0 < showeq.diff
make -f Makefile.dist && ./configure && make && make install

I had a number of problems described here and in other posts. Finally I deleted my entire tree, the execs and started from scratch using the following (exerpted from Adenine! -- thanks Adenine!)

cd /root
md ./linux
export CVSROOT= :pserver:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq
cvs login
(hit enter when it asks for a password)
(If you have never checked out SEQ from CVS: -- obviously true since I deleted my tree)
cvs -z3 co linux
cvs logout
cd linux
(I couldn't get the showeq.diff file using wget, so I used Netscape and saved it to /root/linux)
patch -p0 < showeq.diff
make -f Makefile.dist && ./configure && make && make install

Note, I already had a correct version of libEQ.a in /usr/lib before executing the final line above. As previously posted, you must have the version of libEQ.a that yields an md5sum of
a38ceb68b70e0bc2ea27a5147fa50f73

I now have a working version of SEQ.
