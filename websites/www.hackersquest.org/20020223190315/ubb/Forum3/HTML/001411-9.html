-------------------------------------------

These kinds of errors with autoconf/automake when running the make -f Makefile.dist are basically caused by not having one or more of the proper development tools installed:

m4
autoconf
automake
libtool

Check for these packages to be installed, for example, to check for the m4 package:

rpm -q m4

If it is installed, it would return something like:
m4-1.4-19mdk

If it is NOT installed, you get a message such as:
package m4 is not installed

If any of these packages are not installed, go to http://www.rpmfind.net and in the search box type the name of the package (such as m4, automake, whatever). In the search results, find your linux distribution (Mandrake, RedHat-7.2 for i386, etc.), and click on the .rpm link on the right hand side, and download it. Now install that rpm, e.g.: rpm -ivh m4-1.4.1-5.i386.rpm
If it says the package is already installed, but you have a newer version, then just use the upgrade switch: rpm -Uvh m4-1.4.1-5.i386.rpm. If it complains about dependencies... well, this isn't an RPM thread, lol.

Just GO GO GO and get these packages now, and move beyond make -f Makefile.dist

I just thought I'd add for those installing these, what they are for:

m4: A macro processor, used by autoconf to create configure scripts. See http://www.seindal.dk/rene/gnu/ for more info on GNU m4.

autoconf: Generates your configure scripts. Eases portability, compilation configuration, et. al. Read about autoconf at http://www.gnu.org/software/autoconf/

automake: Used to automatically generate Makefiles. See http://freshmeat.net/projects/automake/ for more info on automake.

libtool: Tool for portability / ease of use of libraries. http://www.gnu.org/software/libtool/

Hey dopie/dragonchomper (heh)

You need libEQ.a in a directory in your library path. copy libEQ.a to /lib or /usr/lib, chmod 644 on it, make sure it's chown root.root, example:

cp libEQ.a /usr/lib
chmod 644 /usr/lib/libEQ.a
chown root.root /usr/lib/libEQ.a

If you want to be able to compile / use libraries in /usr/local/lib, make sure /usr/local/lib is listed in /etc/ld.so.conf and you ran ldconfig after editing it (and do be careful with your ld (dynamic linker) files).

By the way, remember linux is case sensitive, it must be libEQ.a, not any other form of spelling and/or case.

Yes, I know that's for the shared library cache, but the compiler (linker) uses any dirs there to search for libraries, even static libs.
