Zombie 		posted 08-30-2001 09:01 PM            
Although having it open sourced would be nice, I'd like to see a fully functional core first. Along that line, the things I'd like to see (in other words, what would make it a "fully functional core"):
1) Scriptability. The current script syntax is great. Work on it a little bit, and add it to the zone server.

2) Add a "reference" login server. err, the ability to set up all world servers such that they submit necesary info to a central login server when they are set up.

3) Some amount of hard-coded pathing algorithms. Mainly low-level stuff, such as 'go_to x y z', or possibly 'run_from dynel'

4) Support for vendors. Their item/price list would have to be stored in a database somewhere.

Really, when you think about it, those 4 features would allow you to do more or less anything with the emulator. Agro lists, saving/loading, quests, spawn maps, etc etc etc. These are all just scripts, so if you add scripting support, the rest of us could at least work on the whole slew of scripts needed for a fully-functional world.
