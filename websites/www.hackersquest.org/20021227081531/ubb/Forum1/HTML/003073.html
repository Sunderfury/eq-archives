Okay, there are about two or three posts running around about zone teleporting, so I figure we can all post under this one, that way we won't have to jump between each one.

Now, I made an interesting discovery last night involving the adjacent zone thing. (zone names changed just in case) I was in [Neriak], and was trying to find exactly what makes the client think which zone you are in. So I changed every instance of [neriaka] in memory that I could find into [mistmoore], then I set the spell so that it would teleport me to mistmoore. When I cast the spell, it teleported me not to Mistmoore, but back to NeriakA. The significance of this?

Well, it shows that you can trick the client when it involves zones. Nowhere in the spell data did I include the word "Neriaka", I had it set to teleport me to Mistmoore. But, since every instance that it found Mistmoore, it was actually NeriakA, it teleported me to NeriakA. I'm now narrowing down the offsets so I can find a set of one or two to make complete inter-zone teleportation...

Well, there's my two cents. All you other memory editing people out there toss in your ideas, progress, or whatever, and hopefully we can eventually achieve complete inter-zone teleportation...
