Target Audience

This document is intended for those interested in programming templates, and the more technical-minded wiki contributors. This is NOT intended as a how-to for beginning editors, or for those that have never used a Wiki before. If you have never read any documentation intended for programmers, this is probably not for you.

If you are just looking for information on how to use the templates that have been created for your game, see Category:Templates for a list of Template Users Guides.

Also, our Wikibase is at least 90% compatible with the standards created at MediaWiki. You can always use the documentation there, and what they teach you will probably work here.

Wiki Markup

  • Directives - instructs the wiki compiler to do specific things
  • Links - Internal and External linking, including images
  • Text Markup - Styling
  • Tables - Wiki table markup
  • Templates-How To - Syntax, notes, tips and tricks for writing templates.
  • Wiki Table Style Guide - Classes defined for use in all Wikis here

Functions

  • Internal Functions - Those functions defined by the devs and built-in to the Wiki engine
  • Templated Extensions - Templates intended to provide extended capability using the built-in functions.
  • Global Templates - Templates intended for use by more than one game.
  • Template Users Guides - See the subcategories list for documentation of templates designed for a specific game.

Variables

  • Magic Words - These are pre-defined variables you can use in any template. Some are similar to functions.

Color Schemes

  • Bludwyng's Color Scheme Test - This page combines multiple samples showing all the different classes that Bludwyng expects to find in each game's .css file. This page was created as a quick way to see how Bludwyng's classes look against the total .css for each game.
  • Wiki Table Style Guide - How to apply HTML classes and styles in Wiki.
