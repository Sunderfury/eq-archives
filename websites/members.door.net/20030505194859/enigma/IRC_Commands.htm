CHANNEL MODES

Usage: MODE #<channelname> +|-<mode> [<nickname>|<user@host>] 

 

Sets various options on the channel. You must be a channel operator to issue these commands:

 

Mode Description

b Ban: Will ban the specified user from joining the channel.

i Invite: Will make the channel invite only. Users must be specifically invited to join using the /invite command.

k Keyed: Sets the channel so only users who specify the channel key (password) as the second parameter of the join command can join the channel

l Limit: Limits the number of users that can be in the channel simultaneously.

m Moderated: Causes a channel to be moderated so only the operators and those users who are +v (voice) can talk.

n No external messages: No outside messages or notices can be sent in to the channel by users who are not in the channel.

o Gives the user whose nickname immediately follows the +o channel operator status.

p Private: Marks the channel as private. Will not appear when other users do /list command.

r Registered: This indicates that a channel is registered. (informational only) (CR native mode) (users cannot set this mode)

s Secret: Hides the channel in the list

t Sets the channel so that only channel operators can change the topic.

v Gives the user a voice when the channel is moderated (+m)

 

Example: /mode #webchat +s

/mode #webchat +o Surfin (makes nickname Surfin a channel operator on #webchat)

/mode #webchat +b lamer@pm02.digex.net (bans this hostmask)

/mode #webchat +l 30 (limits the channel to 30 users)

/mode #webchat +k ircpass

/mode #webchat +tns

/mode #webchat -sk ircpass

/mode #webchat +kl ircpass 100

/mode #webchat +v Surfin 

 

 

CHANSERV

Usage: ChanServ 

 

ChanServ allows users to register a channel. Using ChanServ allows the channel to be controlled without need of an IRC bot and makes channel takeovers virtually impossible. ChanServ will drop a registered channel if no AOp's or SOp's join the channel for 20 days.

 

ChanServ Commands:

ACCESS Determine the access level of a user in a channel

AKICK Manipulate the AKICK list of a channel

AOP Manipulate the AOp list of a channel

DEOP Remove Operator status on a user in the channel

DROP Drop a registered channel

HELP Provides help on various ChanServ commands.

IDENTIFY Identify as the founder

INFO Get general information about a channel

INVITE Invite a user into the channel

MDEOP Remove Operator status from all users in the channel

MKICK Kick everyone out of the channel

OP Set Operator status on a user in the channel

REGISTER Register a channel

SET Change channel settings

SOP Manipulate the SOp list of a channel

VOP Manipulate the VOp list of a channel

WHY Tells you why a person has OP status in a channel

 

Example: /chanserv help <command>

Usage: ACCESS <channel> [<nick>] 

 

Use the access command to find out what access level you or another user have to a registered channel. ChanServ will return one of the following values:

 

6 Identified as founder of the channel

5 Identified to the founders nick

4 Matches channel founder access list

3 Channel SOp

2 Channel AOp

1 Normal user

-1 AKICK'ed user

NICKSERV

Usage: NickServ <command> 

 

The Nickname services commands are supported in the ConferenceRoom Professional, Developer and Enterprise Editions of the product. The Personal Edition of ConferenceRoom does not support Nickname services. NickServ allows users to register their nicknames for use while online. Users may reserve their favorite online name for their personal use with options to ensure that nobody else can use it. After registration, a user can disconnect users who try to use their nick. If a user does not use a registered nick for the number of days set in the services nickname configuration, the registered nickname expires and becomes available for registration by other users.

 

NickServ Commands:

ACC Check the access of a user to a nick 

ACCESS Change/View your access lists 

DROP Drop registration of your nick 

GHOST Ghost a user who is using your nick 

IDENTIFY Identify to your nickname 

INFO Get information on a registered nick 

LDAP List available LDAP info on a nick 

RECOVER Recover your nickname 

REGISTER Register a nickname

RELEASE Release an enforcer that is holding your nick 

SET Change the settings/options of your nick 

HELP For detailed information: /msg NickServ help <command> 

 

Example: /nickserv help 

 

 

ACC

Usage: ACC <nick> 

 

To check if a user owns a nickname, you use the acc command. After using the acc command, NickServ will reply with an access level that the online user has to the registered nick. The possible values are: 3 User has identified and owns the nick:

2 User matches the nicks access masks 

1 User does not own the nick 

0 User is not online

 

Example: /nickserv acc dalvenjah

SILENCE

Usage: SILENCE <nick> | <user@host> 

 

The silence command allows you to ignore a specific user. For example, if a user is flooding or harassing you, the user can be silenced. When you silence a user, all messages, notices or ctcp events initiated by that user will no longer reach you. Others see this user's messages normally. To use the silence command specify /silence plus a user@host mask or nickname. Any user that matches that mask will not be able to send private messages or notices to you. To remove a silence, issue the same command with a minus sign "-" in front of the hostmask or nick.

 

Example: /silence rowdy

/silence Flooder!trill@pm02.digex.net

/silence Rowdy

/silence *!*@*.digex.net

/silence Rowdy!*@*.digex.net

Usage:     /kline <user@host> <reason>     

 

Will prohibit a user from logging on to your server. This is used normally to restrict access from those that have been disruptive. To unkline someone you must either do it through the control panel or use the /unkline command. You can use wildcards while creating a ban, though you want to be careful of making the kline more prohibitive than needed. The second example is a site ban and will prohibit anyone from that domain from your server. Care should always be used when placing klines.

SHUN

Usage:     SHUN +|- <target>  

 

Allows you to specify a mask for users (in the form user@host) that will not be able to send private messages or notices on that specific server. The rest of the network is unaffected by this command. To remove the shun, just issue the same command with a minus sign "-" in front of the hostmask or nick you wish to unshun. To get a list of shun's on your server type /shun with no mask. Wildcards will work on this command but are not recommended. You should target an exact mask rather than make one over broad that might accidentally shun other users.

 

Example:     /shun Mark!trill@pm02.digex.net  /shun *!*@*.digex.net  /shun -*!*@*.digex.net  /shun

Bots are simulated clients that act as your agent in a channel. Bots have two (2) functions. The first is to keep a channel open so that it will show in the channel list, when you have your server set so that users cannot create channels. The second use for the server bot is to send a welcome message to users who enter the channel. Used by itself it will return all the bots configured to run on the server.

Please Note that the password you set on a bot allows you to identify to this bot in a given channel, make the bot invite you in or give you the op status in a channel.

Commands:

HOST        Sets the hostname for the bot

ADMINSERV CLIENT

Usage:     CLIENT   

 

The client command set lets you add client profiles for the server. A profile is a set of commands that set certain restrictions on users as they try to join the server. You create a profile and add a user@host mask along with a Message of the Day (MOTD) for that profile. ConferenceRoom will match against the first profile it hits, so if you want to add special message for certain hosts the more restrictive profile should be placed before a more open profile. As an example if you allowed all users to join your server (ConferenceRoom default) but wanted to set up a special message for all company employees, you would create a profile called "a-company" with a hostmask set for all the user addresses in your company. CR would see these users as they join and match them against the hostmask in the first profile, if they didn't match it will go to the next until it reached the "z-default" profile which would allow them on.

Commands:

ADD         Adds a client profile to the server.      

DEL         Deletes a specified client profile.   

RENAME      Renames a profile.      

LIST        List the information of a client profile.   

MOTD        Add an MOTD for this client profile.   

MASK        Add or delete a client profile.   

SET         Sets specific client profile options.   

ENABLE      Turns on the profile.   

DISABLE      Disables the profile while saving the information.

This command will restart your services (nickserv/chanserv) while you are online.

 

Example:     /servstart     

 

 

UNAKILL

Usage:     UNAKILL ident@hostname.com  

 

This command has the same effect as /rakill it removes an akill that was placed. This command removed the akill from all the servers.

 

Examples:     /unakill services@webchat.org     

 

 

UNQLINE

Usage:     UNQLINE <name>

 

The purpose of this command is to delete a temporary Qline on the server/network. To list the Qlines you would type /stats q , in this list the temporary Qline is defined by a q, while the permanent Qline is defined by an I.

 

Examples:      /unqline f*ck    

 

 

UNSHUN

Usage:     UNSHUN <name>  

 

Unshun is used to removed a name from the shun list.

 

Examples:     unshun Poolio  

 

 

UNZLINE

Usage:     UNZLINE <host>

 

This command will remove a zline from the /stats k list, the zlines are prefixed by a z.

 

Examples:     /unzline 999.999.99.999

 

 

WALLOPS

Usage:     WALLOPS <message>

 

Wallops is used by Operators in a method of communication. However is a normal user is set to mode +w he/she will be able to read your messages.

 

Examples:     /wallops hey guys :)   

 

 

WATCH/B>

Usage:     WATCH [command]<name>[command]<names>...

 

The watch command is a superior replacement for the ison command. It's designed to let you watch specific nickname and see if the quit, join, change nicknames, are in a server split, come back on line.

You then get instant notifications.

s     shows you who is on your WATCH list 

l     shows you who is on your WATCH list and who is currently online     

L     shows you who is on your WATCH list and who is on online and who is not online 

c     clears your WATCH list 

+/-   add or delete a name 

 

Examples:     /watch +Web +John  (will WATCH for any changes or occurrences on both nicknames)     

 

 

WEBADMIN

Usage:     WEBADMIN

 

Forces the web server to restart.

 

Example:     /webadmin

 

 

ZLINE

Usage:     ZLINE <IP address> or Domain

 

The Zline command is used to prevent a user from making any connection attempts on your server. The Zlines are listed in the /stats k listing, and are preceded by the z letter to let you know it's a Zline.

 

Examples:     /zline 127.0.0.1  /zline shell.webchat.org
