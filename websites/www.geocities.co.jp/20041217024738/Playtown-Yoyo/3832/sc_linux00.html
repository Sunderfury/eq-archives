・SSHの設定('03/12/31)

< 概要 >

SSHの接続方法には、大きく分けて3種類ある。

(1)パスワード認証(デフォルト)

　リモート先のユーザパスワードを認証に使用する。telnetやftpのイメージに近い。

(2)公開鍵認証

　リモートに登録する公開鍵と、ローカルに置き、公開鍵を復号する秘密鍵の2つの鍵を生成し、認証に用いる。併せて作成時に設定したパスフレーズを認証に用いる。

　鍵生成法式としては現在SSH1RSA、SSH2RSA、SSH2DSAの3種があり、後者に行くに従って複雑な認証になる。今回はSSH2DSAを使用する。手順はほぼいっしょ。

(3)ホスト認証

今回は使用しなかったので割愛。

　

< リモートホストの設定(1)：RedHat9 >

(1)鍵の作成

$ ssh-keygen -t dsa

　ファイル名の確認はそのままリターン。パスフレーズを入力する。~/.ssh/id_dsa.pub が公開鍵、 ~/.ssh/id_dsa が秘密鍵。

rsa2の場合は -t rsa
rsa1の場合は -t rsa1 となる。

　

< ローカルホストの設定(1)：RedHat9 >

(1)鍵の作成

リモートホストの作成方法といっしょ。ローカルの鍵と解るように適当にリネームする。

$ cp ~/.ssh/id_dsa.pub ~/.ssh/id_dsa.pub.rocal

(2)SSH：パスワード認証で公開鍵をリモートにコピー

リモートにsftpで接続し公開鍵をコピーする。パスワードは接続先のユーザパスワード。

$ sftp fre2dom@hoge.com
(fre2domのパスワード入力)
sftp > cd .ssh
sftp > put id_dsa.pub.rocal

sshで接続しauthorized_keysを作成。

$ ssh fre2dom@hoge.com
(fre2domのパスワード入力)
fre2dom]$ cd .ssh
fre2dom]$ cat id_dsa.pub.rocal >> authorized_keys
fre2dom]$ chmod 600 authorized_keys

パーミッションの設定を忘れないよう注意。

(3)繋いでみる

$ ssh fre2dom@hoge.com

今度はパスフレーズを入力すると入れるようになる。

(4)ssh-agentの使用

パスフレーズは一般的にユーザーパスワードより長くするので、入力が面倒な場合にはssh-agentを使用する。Xの起動後に、

$ ssh-add

を実行し、パスフレーズを入れると、今後X上ではパスフレーズを聞かれずにリモートに入れるようになる。

　

< ローカルホストの設定(2)：Windows >

Windowsでは、Puttyというクライアントを使用する。設定方法の詳細については以下のリンクを参考に。

http://pure.fan.gr.jp/simm/ssh/putty.html

躓くポイントは3つ。

(1)authorized_keysのパーミッション

(2)keygenを使った公開鍵作成時にコピーアンドペーストでファイルを作ること。文字コードはEUCとし、改行コードはLFのみとすること。最期に改行を入力しておくこと(複数鍵をリモートに登録する時に、次の鍵とつながってしまう)

(3)puttyjpの設定の、SSH-認証-認証のためのプライベートキーファイル指定を忘れないこと

　

< リモートホストの設定(2)：RedHat9 >

無事公開鍵での接続が出来るようになったら、セキュリティの弱いパスワード認証を行なわないように設定を変更する。

$ sudo vi /etc/ssh/sshd_config

< CVSの利用 >

リモートでCVSの設定が済んでいれば、ローカルの ~/.bashrcの末尾に

export CVS_RSH="ssh"
export CVS_ROOT=":ext:fre2dom@hoge.com:/home/fre2dom/CVS_DB"
(:ext:[ホスト名]:[リポジトリパス])

と設定しておくことで、リモートホスト経由のCVSが利用できる。

＜Let'sNote(CF-M2)にVine2.6r1を入れる＞

・インストールについて('03/10/26)

　(参考？)今回は実際にはRH9を入れる予定だったが、CD-ROMブートさせるとテキストインストールしかできない上、途中でCD-ROMが読めない、というエラーを吐いてインストールが中断されるためインストールは見送った。単純にCD-ROMが変だった、という可能性もあるが、一応だめかもしれない、と書いておく（；´Д｀）。Vine2.6r1はグラフィカルブートで立ち上がり全然問題なし。

　今回は、標準でインストールされているWinMeとデュアルブートできるようにインストールを進めた。 リカバリCDでのリカバリ直後は、10G丸ごとWinMeのパーティションとなっているため、未使用部分を分割して別パーティションとする。

　パーティションの分割にはfipsという定番ツールがあるが、どうもFAT32での分割に難がある(*)ようなので、PartitionResizerを使用してみた。
(*)このコンテンツはVine2.1の頃作成されているため、fipsもバージョンアップしてるのかなと思ったが、どうやら2.0のまま変わっていないようだ。

　FDからのブート環境で動かす必要があるため、DOSの起動ディスクを作成した後、ダウンロード後解凍したpresizer.exeをコピーしておく。

　このフロッピーで起動した後、presizerを起動してドライブを2つに分割した。WindowsとLinuxのインストールソフトの量を考えてざっくり4GをWinMeの従来ドライブ、6Gを未使用のLinux用ドライブとした。

　Vine2.6r1のCD-ROMからブートし、インストールを進める。特に引っかかる場所は見当たらず。DiskDruidで自動でHDD割り当てを行い、ブート関連ではliloを使用。ブートローダはMBRにインストール。WinMeをデフォルトブートにするとひょっとすると都合が悪いかもしれない(経験則)。

　インストール後再起動しても、LILOの動作、コンソールの動作、Xの動作も一応問題なし。


・FDDの使用('03/10/26)

　FDDはUSBインターフェースで接続しており、内部ではSCSIのドライブとして認識されている。 マウントには/dev/fd0ではなく、/dev/sdaを使用する。

# mount /dev/sda /mnt/floppy など。

mtoolsを使っている場合は/etc/mtools.confの、

## Linux floppy drives
drive a: file="/dev/fd0"(以下略)

という行のfd0をsdaに置き換えれば、普段どおりmtoolsが使用できる。


・サウンドの設定('03/10/26)

　CF-M2では、NeoMagic 256AVというチップ(ビデオドライバと統合されている?)が使用されている。モジュールはすでに用意されているので、/etc/modules.confに、以下の行を追加する。

alias sound nm256_sound
options nm256_sound force_load=1

ドメイン名：AirH(何でもよい)
サーバー1、2：指定されたプライマリ、セカンダリDNSをそれぞれ設定、3は空白のまま

設定後、接続ボタンで接続を確認した後、AirHという名前で保存。

　WindowMakerのxppxpmドックから簡単に接続できるように、ドックを右クリックして設定を選択、

アプリケーションのパスと引数の文字列の最後に、

xppxpm -wmaker -shape -geometry -O-O AirH(保存したファイル名)

を追加。

・かなり適当な起動手順の解説('03/11/02)

(注)不明なところ、理解不足のところはかなりはしょったり適当に解釈しているとおもわれますので参考程度に（；・∀・）。

# startx

を実行すると、だいたい以下のような順序で起動が行われるようだ。

(1).xinitrcの存在判断

・ホームに.xinitrcが、
ある→xinitを、~/.xinitrcを引数として実行　→ユーザの設定したスクリプトに動作が移る。[end]
ない→xinitを、/etc/X11/xinit/xinitrcを引数として実行

(2)xinitを、/etc/X11/xinit/xinitrcを引数として実行

・Xのリソース読み込み
/etc/X11/Xresourcesがあったら読み込んで更新(xrdb merge)
~/.Xresourcesがあったら読み込んで更新(xrdb merge)
その他、Xmodmap、Xkbmapも似たように処理

・/etc/X11/xinit/xinit.dディレクトリにあるスクリプトをすべて実行
xinputで言語対応などをしている模様

・Xclientsの確認
~/.Xclientsがあったら、~/.Xclientsを実行　→ユーザの設定したスクリプトに動作が移る。[end]
/etc/X11/xinit/Xclientsがあったら、実行
Xclientsがなかったら、xinitでとりあえず立ち上げてネットスケープを実行したり、fvwm2やtwmで表示しようと努力している[end]

(3)/etc/X11/xinit/Xclientsを実行

・/etc/sysconfig/desktopの中身を確認
　中には$DESKTOP=GNOMEやら、KDEやら、WINDOWMAKERなどが書かれている。中身に対応する起動プログラムに動作が移る。ちなみに、 GNOMEはgnome-session、KDEはstartkde、WINDOWMAKERはwmaker。[end]
　desktopがなかったり、中身が違ったりしていた場合は、 ~/.wm_styleを探しにいき、中身を確認する。afterstep系、fvwm系、mwm系など。該当した場合は/usr/X11R6/bin/RumWMというスクリプト動作が移る。内部ではWMに応じて各種設定がされて起動されていく。[end]
　 それもないようなら、xinitでとりあえず立ち上げてネットスケープを実行したり、fvwm2やtwmで表示しようと努力している[end] 　

　

(注)テキストログインを選び、startxを実行したときの動作についての説明なので、Windowログインを選んでいるときにこの動作をするかどうかは定かではない。

　

＜Emacs関連＞

・メニューバーを消す('03/10/26)

メニューバーをめったに使わず、そのために1行損していると感じる人は、

./emacs、Vine2.6r1なら./emacs.elに、

(menu-bar-mode 0)

を追加する。同様にツールバーも消したい人は、(tool-bar-mode 0)を追加する。

　

・DELキーをWindowsっぽい動作に('03/10/26)

インストール後のデフォルトでは、DELキーの動作はBSキーと同じ動作をするため、カーソル上の文字を消す、というWindowsのDELキーと同様の動作をさせるため、

./emacs、Vine2.6r1なら./emacs.elに、

(global-set-key[delete]'delete-char)

を追加する。

　

・定義文字に色を付ける('03/10/26追加)

　 Vine2.6r1ではColor-Mateというものを使用。~/.emacs-color.el内の記述

;(setq color-mate:theme-file　"テーマ名")

が何行かあるところの、いずれかの先頭の;を削除すれば、それぞれのテーマの色分け表示をしてくれる。とはいえ、液晶でまともに使える設定はMidnightくらいか。

　Color-Mateを使用しないディストリビューションでは、~/.emacsに、

(global-font-lock-mode t)

を設定する。

Vine2.6r1の場合は、~/.emacs.elとなる。

　

・Cモードでのインデントの選択('03/10/26追加)

　 ~/.emacsに、

(setq c-default-style "gnu")

という記述を加える。"gnu"にはほかに、"k&r"、"bsd"、"stroustrup"、"whitesmith"、"ellemtel"、"linux"等が記述できる。それぞれのインデントスタイルについては、cc-styles.elを参照。

Vine2.6r1の場合は、~/.emacs.elとなる。

　

・バイナリエディットモード('03/01/03追加)

　ファイルを読み込んだあとで、

M-x hexl-mode

を実行することで、バイナリエディットが可能となる。また、ファイルを読み込む前にhexモードを指定することもできる。この場合は、

M-x　hexl-find-file

を実行して対象のファイルを読み込めばよい。
