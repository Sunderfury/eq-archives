As you may have noticed, I also added recovery times in the right most column.

Note 1
Some skills have delay like casting time on spells. The actual cycle time is duration + recovery time, so you cannot bind every 10 seconds.

Note 2
The client (your PC) refreshs, for example, the bash button in exact 8.0 real world seconds after you hit the button no matter how much lag you get. However, the server won't let you bash twice in 8.0 frame-seconds which can get greatly longer than 8.0 real world seconds when the load average on the server exceeds 100%.
This implies that you could waste a chance to bash if you hit the button 8.2 seconds after the last bash while the server thinks it has only been 7.5 frame-seconds. At this point, the bash operation is refused and the client would lock the button down for another 8 seconds.
