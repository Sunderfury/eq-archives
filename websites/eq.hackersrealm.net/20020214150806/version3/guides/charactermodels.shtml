Character Models Guide
By: jellognome
Print This Guide


Using EQInside, open up chequip.s3d.


chequip.s3d contains the models of all the gods, all the players, all the elementals, the lich, the skeleton, and the werewolf. For players, the first two letters represent race. The third letter represents sex. ELF = woodelf female. OGM = ogre male.


each model has 5 pieces, and here are the TAGs: XXXHE00_DMSPRITEDEF, XXXHE01_DMSPRITEDEF, XXXHE02_DMSPRITEDEF, XXXHE03_DMSPRITEDEF,

XXX_DMSPRITEDEF.


They can be found in any order in the file so don't expect something simple. XXX_DMSPRITEDEF is the body file. It has no head. XXXHE00_DMSPRITEDEF is the helmetless head information. HE01 is for leather, HE02 is for chain, and HE03 is for plate. New helmets from Velious are not stored in this file.


Now let's take a look at the actual vertices and faces. In OGM.SPK under the body information you will see


NUMVERTICES 625

XYZ -1.3381259e+000 -1.2489295e+000 1.5096866e-001

XYZ -1.3361061e+000 1.3277209e-003 3.7859764e-002

XYZ -9.1086037e-002 -1.5801769e+000 1.7924576e-001


This represents the first three vertices of the Ogre's body. Each one is invisibled numbered, but you don't see that right now.


Moving down the .SPK file you'll get to the UVs. This defines all the information about how the textures are applied. Keep scrolling and eventually you'll come to this:


DMFACE2 //1

TRIANGLE 52, 51, 50

ENDDMFACE2 //1


This connects vertices 52, 51, and 50. Everything is made of triangles because any 3 points can be connected by a plane.


.MDF means material definition file, and it displays the material accordingly. The file called XXX_alt.mdf compiles all the MDF files into one useful file which actually paints the object so you can have mismatched armor, like leather bracers and plate leggings.


I believe MESHOP contains information about the skeletal structure, how the model bends. Also referred to as bones.


The files called c01XXX.sph - c10XXX.sph, and d01XXX.sph to d05XXX.sph contain all the data about animations. Keep in mind that each file does one animation, and there are more... l01XXX, p01XXX, etc.


So what does it all mean? Well, with the vertices alone you could easily write a program to convert this into another 3D format. The most versatile formats are probably the .X format, supported by Microsoft, and the .3DS format which is the standard for 3DStudio Max. Unfortunately both of these files are in hex and can not be read so easily by a text viewer. The next most user friendly format is the .ASC file format. To attune the wireframe to ASC standards, the ogre becomes:


Named Object: "OGM_DMSPRITEDEF"

Tri-mesh, Vertices: 625 Faces: 326

Vertex list:


Vertex 0: X: -1.3381259 Y: -1.2489295 Z: 0.15096866e

Vertex 1: X: -1.3361061 Y: .0013277 Z: .0378597


etc etc, then:

Face 0: A:52 B:51 C:50 AB:1 BC:1 CA:0

Smoothing: 1

Face 1: A:55 B:54 C:53 AB:1 BC:1 CA:0

Smoothing: 1


Smoothing groups can be preserved.
