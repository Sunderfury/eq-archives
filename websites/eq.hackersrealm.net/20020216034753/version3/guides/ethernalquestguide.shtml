1.1 HackersQuest Information

HackersQuest is responsible for the creation of this emulator and all of its underground fame.

V1.0Beta. If you are using a different version of the Emulator, some of this information may not work for you.


1.2 System Specs

While there is very little information on the actual system requirements for EthernalQuest, through the forums it has been

posted:


Pentium 3 500mhz

64 - 128 Megs of SDRAM

8 Meg 3d Video Card or Accelerator

200 megs of Free Space (Cache)

High Bandwith Connection


The emulator has also reported run well on these specs:


Pentium 2 400mhz

128 Megs of SDRAM

8 Meg Video Card

High Bandwith Connection (Cable Modem)


This will allow you to run EthernalQuest solo, with just the server's player connected with no lag at all. While these figures

are still sketchy, its the only thing that we have to go off of by now. If anyone sucessfully runs the Emulator with different

hardware, please send me a e-mail to Alvec@home.com detailing the speeds and hardware.


The emulator ALONE can apparently be ran on a computer as simple as this:

To run the emu you need nothing more than a 486 with 32megs of ram (perhaps less)...It requires just a vga adapter.

2.1 Installation

0) Back up your Entire EQ directory to a another spot, that you will only be using for EthernalQuest.
1) Download EthernalQuest (Emu Server)
2) Unzip to your fake Eq dir (Containing ALL updated files)
3) Patch the eqgame.exe and .dll with these files- http://alvec.hypermart.net/eqfiles.zip
4) When you opened Ethernal quest zip, did you see a zip file called Emulog.zip (it is Inside the Emu server zip)
5) Unpack that Emulog.zip into the FakeEQ dir
6) Kill all your .OPT files and Client.ini
7) Run all your Server scripts (Loginserver, worldserver, zoneserver)
8) THEN run Emulog.exe

(MAJOR NOTE: NOT Emulogin.exe, which is on the Hackerquest page for download, only use the Emulog.exe

that came with the Server package zip) Doublecheck this, most common error people are making is this part.


2.2 Patching

Like it states in the installation procedure, it is important to patch everquest with the files sent off of the Verant server

to begin with. Once that is finished, the files needed to play EthernalEQ are located at the http://alvec.hypermart.net/eqfiles.zip

It is important to know that if Everquest.exe is run again, then the files in the EthernalEQ folder will be overwritten and

the Emulator will not work again until it is patched with the proper files.


2.3 Troubleshooting

In all my project experience, I have never seen so many different errors pop up in different situations. And while some

of these fixes may work for some people, some still have problems and additional errors.


#1- This is a very common problem, As was stated in 2.1:

(MAJOR NOTE: NOT Emulogin.exe, which is on the Hackerquest page for download, only use the Emulog.exe that

came with the Server package zip) Doublecheck this, most common error people are making is this part.


#2- If you recieve the following message while running Emulog.exe:

File = 1830912 | cmp = 2140160 (Note: The #'s could vary)

This means that you do not have the up to date files required to run the emulator. The files must be patched with http://alvec.hypermart.net/eqfiles.zip


#3- While running the emulator, you recieve this error:

A required .DLL file, *********.DLL, was not found

This means that your Visual Basic Runtime files are out of date. A set of newer files can be currently downloaded at

http://alvec.hypermart.net/VB6.zip


#4- Simple error with a simple fix.

Error opening packed file system (eqfeart3.pak)

This means that you did not patch everquest before you applied the required emulator files. Patch everquest normally, and

then apply http://alvec.hypermart.net/eqfiles.zip


#5- Illegal Operations:

Definately the nastiest and less understood of all the errors, there is no set fix for this problem. This type of problem can be

caused by many things. The simplest thing to do is try to re-install Everquest and EthernalQuest. If that doesn't work, you

may have to re-install Windows. And lastly, the thing that everyone hates to hear, you may have to format your hard drive

and then re-install everything from there.


#6- The Blank Error Window

If you are encountering a empty error window, you have to search for glide*.* files on your computer. Delete all of these files

that appear and then it should work like a charm.


And as *Drakeimperor* did point out,it is IMPORTANT to keep directory structure in order for all of the files to work!!!




3.1 Server Information

Due to the large # of questions about server capabilites, the following list has been organized:

There is currently NO combat in Ethernalquest. This includes fighting npc's and any form of PVP.

There is currently NO spells in Ethernalquest. This includes anything player casted or npc casted.

The Item database is not 100% complete.

Zoning under normal procedures has not been implented. You must use the #zone command and then pick a location

from the zone database (See 3.3 for more information on this).

The only character you can create will always be a Level 50 Dark Elf Enchanter.

For a listing of ingame commands up the #help command.

And will everyone please remember that the server is still in Beta stages.


3.2 Connecting to Remote Servers

To log into someone elses server:

Go to your Everquest dir. Then look for the IP.ini. Open it and you will see something like this.

Example:

Server IP|Port
127.0.0.1|5999|

Now you need to change the IP and port to the numbers of the server you want to access like so. Say your friends IP is 234.456.23.69 and port is 9000.

Example:

127.0.0.1|5999| local ip

*Change to friends server IP and Port*

234.456.23.69|9000|

Now save the file and click on Emulog.exe in the Everquest dir and you will now be set to access your friends IP.

(Warning) Make sure to backup your old IP.ini as IP.bak so you dont forget the numbers for your local server.

Have a nice day =)


3.3 Zone Information

#1. Zoning is still not fully implemented. There are currently 2 ways to zone. The first was is to use the #zone command and

then type in the name of the zone. An example of this would be:

#zone qeynos2

And in doing that command you would be sent to Northern Qeynos.


#2. There is also another way to fix which zone boots at the Emulator startup.

Open the Scripts subfolder in your EthernalEQ folder. There you will see a file named ZoneServer.js

Open that file in notepad, and the 4th line will look like this

SetZone("qeynos")

now you can change qeynos to any zone you want in the game (except for the buggy ones, and if someone wants to post

those it would be appreciated). If you need help with the names of the zones, a listing can be found in the Zone subfolder in

the main EthernalEQ folder. All the zone names are listed there in the filenames. So if you wanted the server to boot in

Qeynos hills for example, you would edit the ZoneServer.js script to say on the 4th line:

SetZone("Qeytoqrg")

(Note: A Zone listing can be found in section your zone subfolder, or in section 6.1)


3.4 Spawn Information

Currently the only way known to spawn mobs is the #spawn command.

To spawn a mob:

type #spawn #1 #2

#1 will be the number of the mob for example #14 will spawn a were wolf

#2 will be an item the spawned npc will be wielding. If you do not with to give the spawned creature anything simply enter 0

So thus if one wanted to summon an iksar skeleton wielding a fiery avenger they would type

#spawn 161 62

Hope this helps.

(Note: See 6.2 for a spawn listing, and 6.3 for a item listing)

3.5 Hex Editing

I myself have no idea about Hex editing but thankfully someone did submit some resources to me =)

The #zone command IS disabled by default, so you have to do some hex editing to enable it.


A decent Windows HEX editor is Ultraedit. It can be obtained at http://www.ultraedit.com


Now, *1010101* has some instructions on how to edit the #zone command to make it work:


Ok. To enable the #zone command, which will allow you to change zones while in the emulator, do the following:
First, open up ZoneServer.exe with a hex editor. It is located in your EthernalQuest folder.

Now, go to offset 00084364 (0001498C in hex) or do a case-sensitive search for "Ashran"

Next, overwrite "Ashran" with six null characters (00's) and save it.

Now, the next time you restart the server software and run it, you'll log into whatever zone you have it set for normally. Once in, type "#zone zonename" where zonename is the short name for the zone you want. You'll see a message saying you issued a gm command to change the zone. Now /q and log back in. You'll be in the new zone. This does NOT change the zone permanently (doesn't save to ZoneServer.js), so if you shut down the zone server and restart it, it'll revert back to whatever is set in ZoneServer.js
