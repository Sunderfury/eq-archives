Known issues :

Please read this issues VERY carefully before you sending emails to me about bug reports (which are not really bugs), or complains.

  • If EQP stops after parsing line 1 of your log file, chances are you have used some editor to edit and save your log file, taking away the line feed and return from the end of each line. Just open the log in EQP, and save it once, then it will be fixed and ready to be parsed.
  • Since EQ logs everything you see when you play EQ, sometimes messages could get scrambled up and end up logging some garbage characters including "End of file" (EOF) character. If your log have such a character before the real end of file, EQP may stop before it completely parse the whole log, so just remove that line and parse again.
  • By default, EQP uses color parsing and normal speed mode, please change setting if you find that speed to be too slow for you. The fastest speed combination is non-color parsing and speed mode.
  • For best results, please kill a mob one at a time, that way the parser will be able to generate the most accurate result of your ability to kill mobs.

How to use :

In EQ, use /log to toggle chat logging on/off, it will save file to eqlog.txt in your EQ dir, open that file in EQ parser and analyze it.

Note : For best results, please kill a mob one at a time, that way the parser will be able to generate the most accurate result of your ability to kill mobs.
