    Re: Ubuntu Unity

    This just became my favorite feature of Emacs:
    http://sachachua.com/blog/2013/02/re...ling-packages/

    Spaced repetition is a great technique for prioritizing and learning things. Emacs Org has an org-drill modulethat implements flashcards for Org headings that are tagged with :drill:. I’ve been using this to prepare for the Canadian citizenship exam, and that’s been working out well. So I started creating an emacs-drill.org file with headings for the things I’m interested in learning, such as multiple-cursor-mode and ace-jump-mode…
    … And then I remembered, hey, my Emacs configuration is one big Org file (thanks to the joys of literate programming) so why don’t I just stick the :drill: tag onto the relevant snippets and run M-x org-drill there?
    This might actually work out. =)
    This changes everything. Again.

    On HN, I saw a link about Light Table
    Which was a Kick Starter project: http://www.kickstarter.com/projects/ibdknox/light-table

    It looks very cool and smart. It looks like Light Table is aiming to be an Emacs for mortals, as it embraces the notion of being a platform instead of just an editor and the LISP concepts of a REPL as well as Emacs' Comint idea that allows for async process communication between external process and the editor, Emacs most powerful feature perhaps.

    I haven't seen anything like Light Table's real time debugger concept in another program. That is amazingly powerful and would pay major dividends on productivity. I like that Light Table uses JS as its extension language; that is something that a lot of people are already familiar with, and Light Table also uses an already existing editor as its basis for that instead of coming up with yet another editor with its own non-transferable eccentricities.
