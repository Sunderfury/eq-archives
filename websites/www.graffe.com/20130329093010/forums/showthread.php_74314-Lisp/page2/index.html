    Re: Lisp

    Nadiar, some schools use Python as an introductory language. I think MIT does at present instead of using Scheme. There are benefits to both directions, starting from close to how computers work and expanding into more abstracted implementations where pointers and memory management are not directly worked with or going in the opposite direction. The more abstracted languages are better for learning general concepts right away instead of being bogged down in minutiae. Java and C++ seem to be the most commonly used in intro programming classes, both of which are massive languages but commonly used in the real world. I would start people out with python and data structures probably, not sure how much value there is in exposing people to the more bare bones of computing if they are in CS, especially since most end up hating it (dealing with pointers and such).

    I can be kind of terrible about keeping a specific style for brace placement. I tend to put them in the area that makes the code the most readable.

    Often, I may end up with something like this:
    Code:
    #print all items from the list to stdout
for thing in list { print thing; }
    The only time it would get a second piece of code to execute is if I needed to do something else as well, but which was equally small. The most complicated single line I ever write is in BASH scripts, I have a chunk of code that allows me to multithread portions of code.

    Code:
    #if the number of jobs from this script is 10 or more, wait one second and check the number of jobs again
joblist=($(jobs -p)); while (( ${#joblist[*]} >= 10 )); do sleep 1 ; joblist=($(jobs -p)) ;done
    It's only a single line because I use it pretty damn often when I'm writing multi-threaded BASH scripts, and since this already has to be inside of at least one loop, doing the multi-line version can just look distracting to me.
