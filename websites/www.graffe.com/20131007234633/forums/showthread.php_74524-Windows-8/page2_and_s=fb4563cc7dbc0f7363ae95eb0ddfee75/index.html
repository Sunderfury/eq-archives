    Re: Windows 8

    Originally Posted by Bonlainy
    Not it's not complicated, but it's pretty fucking non-intuitive. Why the FUCK would I go to settings to shutdown? Personally that makes zero sense, shutdown shouldn't be hidden under a Settings menu.
    This has to do with the unification and standardization of certain icon functional concepts in Windows. "..." now is always a button where clicking on it gives you additional menu items that are not currently visible, and the Gear icon is used for access to account and advanced settings. I suppose I can agree that putting it there doesn't make the most sense, but considering that it lives with the following set of icons, it does:

    Network, Volume, Brightness
    SysTray, Power, Language

    Change PC Settings



    Also, the only other options on the Start Bar are Search, Share, Start, Devices, and Settings. It might make sense to put it in Devices, since the machine is a device, but at least it's better than clicking Start to Shutdown your machine

    EDIT: Also, why do you shut your computer down anyway? Doing that causes it to perform its maintenance and patching while you're trying to use it. At least once or twice a month you should leave it on all night at minimum, but always-on is really better, because then you get your virus updates, and all patches as soon as they're available, and the system will perform on-line optimization of your storage system during low use cycles (I.E. all while you sleep).

    Re: Windows 8

    Originally Posted by Pyron
    How does Windows 8 work with DAW software and dual screens? I'm tempted to give it a spin since its shiny and different, but I'm wondering how well the new UI works with something like Reaper.
    It still has the same functionality for multiple screens that Windows 7 had, and in particular I'm talking about the WinKey + P functionality where you can select to duplicate, extend, or disconnect additional screens. The display controls for arranging them are buried slightly deeper than they used to be because of the top level UI changes, though, but Right-click Desktop -> Properties is still the correct starting point.

    If your software worked on Win7, and was developed fairly recently, like .NET 2.0 or newer, then you shouldn't have any problem with it on Win8.
