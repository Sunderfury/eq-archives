    Windows 7 no response to inbound IP requests

    I'm stumped trying to figure out why my desktop is not responding to incoming IP requests. Everything works from the desktop, I can browse other computers on the network, print over the network, access the internet, but I can't get anything to connect to my desktop. Other computers (Windows 7 and Vista) can't browse my fileshares, can't connect to my web server and I can't remote desktop.

    It's like there's a firewall enabled that is blocking all incoming requests, and it just started happening last week after having no issues since windows 7 install.

    Things I've done to diagnose it:
    - Disabled all Windows Firewall software in domain, private and public profiles. I have not installed any other firewall software.
    - There are two ethernet ports on my motherboard and I've tried both and have the same result
    - Hooked it up to a totally separate router I got out of the closet. Totally new IP, and I can ping my laptop from desktop but not vice versa.
    - I've done a full Microsoft Security Essentials virus scan and downloaded Eset Nod32 trial and did a full scan. No threats found.
    - Powered down the desktop / multiple reboots. Also booted into safe mode with networking, same result.
    - Checked for new ethernet drivers.
    - Viewing network traffic in Wireshark, when I ping from the laptop I can see an ARP request from the router and response from my desktop, but I do not see the subsequent ICMP packet from the laptop.

    The only thing I've done lately that I can think of was install uTorrent a week or two ago. I got it directly from the uTorrent website, so I don't think it had malicious software. I uninstalled it but got the same result.

    I kind of wonder if it could be hardware related, and want to try a separate ethernet adapter. I need to dig out my USB ethernet adapter.
