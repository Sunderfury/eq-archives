ただ、この表を見てみればわかりますが、場所によって対象によって、さらにSpellやSongからも値に相当の変動があることから、相対的なものが占める割合が高いと言えますね。だいたいこんなモンだよ、と目安程度に留めましょう。

計算してみよ～。
まずは平均DMG。基本となる式は以下。

最大DMG = (( Skill + STR ) / 100 ) x DMG + LevelBonus + 1
最小DMG = LevelBonus + 1

平均DMGは最大DMGと最小DMGを加算して2で割ってしまえばよろしいですね。
式に数値を代入すると、最大DMGは51(小数点以下は切り捨てナリ)、最小DMGは12ということで、足して2で割って、平均DMGは31.5と出ましたよ。さぁどうだっ。

誤差は約-7％。どんなもんだろ。ただ、計算ではSTRは184にしてあり、実際はさらに上昇することもあったので、誤差は少しだけ大きくなるかな。

興味深い形を描いていると思いませんか？
煙突のように突出している2本の棒。これは片方は最小ダメージ値であり、もう一つはいわば、武器による最大ダメージ値とでも言えるものです。これは実は、

最大武器ダメージ値 = 武器DMG x 2 + LevelBonus

といった計算で導き出されます。
通常ならば全体的になだらかな、平面的なグラフになって然るべきものなのですが、何故こういったグラフになるのでしょうか。
これはおそらく、ダメージの計算式には、複数の計算式が絡み合っているからだと思います。
それには、計算にはとても含むことのできない、対象との相対的Level差という要素が入っているのではないでしょうか。まったくの推測にはなりますが、対象が弱い、すなわち相対的Level差が大きい場合は最大武器ダメージ値が大きく反映され、もう一つの突出した値である最小ダメージ値が出にくくなる、と。逆に相対的Level差が小さい場合は、最小ダメージ値に偏りが生じるようになるのだと思います。
そして、これはあくまでも/conにして青もしくは緑、つまり対象のほうがLevelが低い際の場合であり、/conが赤といった、対象のほうがLevelが高い場合には、最小ダメージ値ばかりが目立つようになると予想されますね。

さっそく、Primaryから計算してみましょう。
まずは平均DMGを出すため、最大DMGを計算します。また、EffectによるATK+30の効果は、STRと同様な効果を持つとして考えます。

( 182 + 30 + 250 ) / 100 x 14 + 11.7 + 1 = 77 (小数点以下切り捨て)

最小DMGは12なので、平均DMGは44.5。これをDLYで割ってダメージレートを求めます。

44.5 / 2.4 = 18.5

Double Attack発動率61％と、Runebranded GirdleのHaste率27％を掛け合わせます。

18.5 x ( 1 + 0.61 ) x ( 1 + 0.27 ) = 37.9

最後に実測値からの命中率64.1％を掛け、

37.9 x 0.641 = 24.3 [dmg/sec]


Secondaryを求めます。
最大DMGは、

( 182 + 30 + 250 ) / 100 x 14 + 1 = 65 (小数点以下切り捨て)

最小DMGは1なので、平均DMGは33.0です。DLYで割ります。

33.0 / 2.4 = 13.8

そして、Double Attack発動率61％、Dual Wield発動率61％、Haste率27％を掛け合わせます。

13.8 x ( 1 + 0.61 ) x 0.61 x ( 1 + 0.27 ) = 17.1

最後に命中率64.1％を掛け、

17.1 x 0.641 = 11.0 [dmg/sec]


よって平均DMGは、

( 44.5 + 33.0 ) / 2 = 38.8

実測値38.3との誤差は-1.3％。そしてdmg/secは、

24.3 + 11.0 = 35.3 [dmg/sec]
