1. The Offsets.dat Format

The Offsets.dat file is located in you EverHack/Data directory. You just open it with a simple editor like Notepad to edit the File. You may see something like this:
 

6252724 'Offset of the Target Pointer 
8083704 'Offset of the Pointer to youself
8077064 'Offset of the first Pointer to you Skillist
6254484 'Offset to the first Pointer of you Spellist
116 'This value determines how much bytes everhack have to add to you Self-Def to get the Itemslist.
All values are in bytes, the last value normally don't need to be changed.

2. How to get Target Pointer Offset

All you need to get this Offset is Notepad (to edit the offsets.dat) and an memory patcher like WinHack2 (I recommened it). Of course you need the chance to Alt+Tab out and get back to EQ!

First you go in a city and select any merchant or guard (one which name is unique in that zone) as your target. Than you Alt+Tab out and go into WinHack (or the memory patcher you use). Than you seek for the whole String(Name) of you Target, you must use correct spelling (Upper/Lowercase). Sometimes you might find more than one result. Not a Problem just check the Results again and most of them have now changed to something other. This is cause things in Zone move. If you now have the Adress, something like 24A234E4. Than you mirror the Adress to E434A224 and seek for it. I don't know for sure if EverHack can do this, if not you have to find another way, by making a value out of the hex value and seek for it. Now you should find it below 900000 (Hex). Use the Adress you got now and copy it into calc(Mode Hex) and convert it to a decimal (Mode Dec). Than copy the decimal into the offsets.dat and replace it with the one there allready is. Or jsut create a new file. Its in you hands. Thats all!

3. How to get Pointer to youself

Its as simple (hehe simple for you too?) as getting the Pointer to you Target. If you dont understand what I write here, damn not the correct spelling or grammer, just read in point 2 what there stands. Its more detail!

You just seek in the memory for you own name, copy the adress and mirror it. Than seek for this value again in memory and you should find it below 900000 (Hex). Again copy the adress into calc.exe and convert in decimal. Finally write it in offsets.dat

4. How to get first spell Pointer

As in point 2 and 3 you seek for the Skillname. But beware you need to seek for the first skil (1H Blunt it should be). Than take the adress of it mirror it and seek for the mirrored value. Than you should again find a adress below 900000 (Hex). Convert t with calc.exe into decimals and write it down into offsets.dat

5. How to get first spell Pointer

Like in point 4 you seek for the first spell that exists, damn I really don't remeber it. Mmhhh take a look into the spells.dat in EQ directory and look there for first spell. Than you do all the steps like in point 4, just seek for the spellname!
